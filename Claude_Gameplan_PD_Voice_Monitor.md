## 1. What the Science Says (Your Pitch Ammunition)

### Why Voice Works for PD
Parkinson's affects the vocal system early — **up to 90% of PD patients** develop speech/voice disorders. Voice changes can appear **before motor symptoms are clinically visible** (prodromal phase). Key insight: voice deterioration follows a partially independent trajectory from general motor decline, making it a unique monitoring signal.

### The Voice Features That Matter

| Feature | What It Captures | Why It Matters for PD |
|---------|-----------------|----------------------|
| **Jitter** | Pitch instability (cycle-to-cycle frequency variation) | Vocal fold rigidity/tremor |
| **Shimmer** | Volume instability (cycle-to-cycle amplitude variation) | Reduced laryngeal muscle control |
| **HNR** (Harmonics-to-Noise) | Voice clarity vs breathiness | Incomplete vocal fold closure |
| **F0 (fundamental frequency) stats** | Pitch mean, std dev, range | Monotone speech = reduced F0 variability — **earliest PD voice marker** |
| **MFCCs** (13 coefficients + deltas) | Spectral shape of speech | Captures overall vocal tract changes |
| **Formants** (F1-F4) | Vowel articulation precision | Reduced vowel space = articulatory decline |

### Progression Pattern
1. **Earliest**: F0 variability drops (monotone speech)
2. **Early-Mid**: Jitter, shimmer increase; HNR decreases
3. **Mid-Late**: Formant space collapses (slurred vowels)

### Accuracy You Can Cite
- Max Little (2009): 91.4% classification from sustained vowels
- Self-supervised models (2024-2025): AUC 0.85-0.95 within-dataset
- Honest cross-dataset benchmark (Adnan 2025): 88.9% AUROC internal, 77.4% on unseen speech
- Multimodal (voice + motor): AUC 0.86+ (2025 Taiwan study)

### Competitive Angle
Existing FDA-cleared PD apps (StrivePD, Parky, NeuroRPM) are all **Apple Watch tremor/dyskinesia monitors**. None lead with voice biomarkers. Voice is non-invasive, requires no wearable hardware, and captures a different dimension of disease than motion sensors.

## 3. Build Plan (48-Hour Breakdown)

### Phase 1: ML Pipeline (~8 hours)

**Step 1.1: Get the data**
- Download **UCI Parkinson's Dataset** (easiest, immediate access, 195 voice recordings with 22 features pre-extracted): https://archive.ics.uci.edu/dataset/174/parkinsons
- Also download **UCI Parkinson's Telemonitoring Dataset** (5,875 recordings with UPDRS scores): https://archive.ics.uci.edu/dataset/189/parkinsons+telemonitoring
- If time: Access **mPower** via Synapse (requires account approval — apply early)

**Step 1.3: Train XGBoost model**
- Binary classification: PD vs healthy control
- Severity regression: predict UPDRS motor score from voice
- Use 5-fold cross-validation, report AUC and accuracy
- Save model with `joblib` or `pickle`

**Target**: AUC > 0.80, accuracy > 75% on held-out test set

## 4. Speech Tasks (What You Show the User)

### Task 1: Sustained Vowel /a/
- Instruction: "Take a deep breath and say 'ahhh' at a comfortable pitch for as long as you can"
- Duration: 5 seconds minimum
- Captures: Jitter, shimmer, HNR, F0 stability
- Why: Most validated single task for PD detection (Max Little's original work)

### Task 2: Reading Passage
- Use the **Rainbow Passage** (standard in speech pathology):
  > "When the sunlight strikes raindrops in the air, they act as a prism and form a rainbow..."
- Captures: Prosody (F0 variability), articulation rate, pause patterns
- Why: Best for severity assessment and monitoring natural speech changes

### Task 3: Diadochokinetic (DDK) "pa-ta-ka"
- Instruction: "Repeat 'pa-ta-ka' as quickly and clearly as you can"
- Duration: 10 seconds
- Captures: Articulatory speed and regularity, oral motor coordination
- Why: Sensitive to bradykinesia affecting speech muscles; tracks non-dopaminergic progression

## 5. Progression Monitoring Logic

### Personal Baseline Approach
Rather than comparing to a population average, compare each patient to **themselves**:

1. **Baseline period**: First 3 sessions establish personal baseline for each feature
2. **Deviation scoring**: Each subsequent session scored as % deviation from personal baseline
3. **Composite PD Voice Index (PDVI)**:
   - Weighted average of normalized feature deviations
   - Weights from XGBoost feature importance (data-driven)
   - Range: 0 (no change from baseline) to 100 (severe deterioration)
4. **Trend detection**: Flag if 3+ consecutive sessions show >10% decline in any key feature
5. **Medication-aware**: Group recordings by on-state vs off-state (based on logged medication times) and track trends separately

### What to Show on Dashboard
- **Session Score**: "Today your voice score is 34/100" with color coding (green/yellow/red)
- **Trend Arrow**: Stable / Improving / Declining over past 2 weeks
- **Feature Heatmap**: Which specific features are changing (e.g., "Your pitch variability has decreased 15% from baseline")
- **Medication Response**: "Your voice scores are 20% better within 2 hours of medication" (comparing on/off state recordings)

## 6. Critical Technical Pitfalls for Hackathon

| Pitfall | Quick Fix for Hackathon |
|---------|------------------------|
| Browser audio is WebM/Opus, not WAV | Use `pydub` or `ffmpeg` on backend to convert to WAV before feature extraction |
| Different browsers record at different sample rates | Resample everything to 16kHz on backend with `librosa.resample()` |
| Background noise ruins jitter/shimmer | Add a simple RMS noise check; show "too noisy, try again" warning |
| Model trained on sustained vowels, tested on reading passages | Train separate models per task type OR stick to sustained vowel for scoring |
| WebM MediaRecorder might not work in Safari | Use `navigator.mediaDevices.getUserMedia` with fallback; test in Chrome primarily |
| Feature extraction is slow | Pre-compute features async; show loading spinner; cache results |
| Small UCI dataset might overfit | Use cross-validation; report honest metrics; acknowledge limitation in presentation |
